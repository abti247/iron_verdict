# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in Iron Verdict, please report it responsibly.

**Email:** iron.verdict.judging@gmail.com

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact

**Do not** open a public GitHub issue for security vulnerabilities.

## Response Timeline

- **Acknowledgment:** within 48 hours
- **Initial assessment:** within 7 days
- **Fix or mitigation:** depends on severity, but we aim to address critical issues promptly

## Scope

The following are in scope for security reports:

- WebSocket message handling and session management
- Authentication or authorization bypasses
- Cross-site scripting (XSS) or injection vulnerabilities
- Denial of service through application logic
- Sensitive data exposure

Out of scope:

- Vulnerabilities in third-party dependencies (report these upstream)
- Issues requiring physical access to the server
- Social engineering attacks
